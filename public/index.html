My first Rails application

Some notes from Stefan on how I got started.

  1. I installed Homebrew, RVM, Ruby, Rails, Git, and Heroku

    This Ruby on Rails Tutorial is the best tutorial I've found so far. The site might be better than the book since it has links to other sections for troubleshooting, etc.

  2. Getting Ruby on Rails installed was a bit of a chore

    Take a look at this how-to: Installing Ruby, Rails, Homebrew, RVM, and Mysql on OSX Lion. I initially struggled to install MySQL on OSX 10.6.4 (hint: you must use the 64 bit version of Mysql), and ended up using OSX 10.7, with sqlite3 as the database. If you plan to host your site on Heroku (and why not?) you can skip the Mysql part.

  3. I took some Rails lessons from a Zombie

    Railsforzombies.com helped me begin to grasp the model, view, controller setup. I've heard the term many times, but until I was going through some examples in the tutorials, it was hard to grasp. I recommend completing the first three lessons before you start Chapter 2. Demo App in the Rails Tutorial (above). The Zombie tutorial taught me things like: how to validate that user names and email addresses are required, and must be unique. I am looking forward to playing around more here.

  4. I configured my accounts on Git (version control and open source community) and Heroku (free hosting) to work from the command line

    Github.com and Heroku.com. Heroku needs a couple special tweaks to your Gemfile, so that your site can use the Postgresql database, which Heroku needs. This is covered in the Rails Tutorial book.

  5. I pushed

    I was a bit nervous, but with my Rails Tutorial at my side, I managed to follow the directions and push the app to Git and then to Heroku. Everything worked, except when I tried to migrate the database to Heroku, I got an error. So, I went back and did a bunch of searches on the error message and landed on this Q&A in Stackoverflow.com (this site is now my best friend) where others had troubleshooted the error, and had posted answers on how to fix. I managed to make the Heroku-specific edits to the Gemfile, and boom. The app was up.

  6. To come

    I'm going to hookup Bootstrap (the CSS framework) and LESS (a tool that let's you handle CSS in a more centralized way) and then once I get that working. I plan to scrap everything, read more of the Rails Tutorial, and conceive of something that might actually be useful, maybe.
